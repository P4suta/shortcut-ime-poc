## 含意（補完 UX と索引設計への反映）
- **実現性 GO**：打鍵削減◎＋中長キーのランキング良好＋72.5%単一候補。データは前向き。
- **構造的発見**：最悪ランクの語（今日 `ky`、愛 `a`）は「**完全な子音スケルトン**」で、子音をもう足せない＝段階的絞り込みが効かない。子音が残る語（共有 `kyy`、駅 `ek`）は上位で良好。→ 短い完全スケルトンほどランクが悪く、かつ子音では絞れない、という相関。
- **決定1（索引設計を左右）：オプション母音**。中国語簡拼の bj / beij / beijing 方式の日本語版。`ky` でも `kyou` でも今日が出る。混んだ短キーは母音を足して絞る。→ 索引を「子音のみ `FrozenDictionary`」から **ローマ字トライ（子音必須・母音オプション走査）** へ。**トライ実装前に確定**（後付けは作り直し）。
- **決定2：学習（recency）を第一級**。子音で絞れない短キー（今日・愛）の主要な曖昧性解消。1回選べば次からトップ。「二次的な磨き」ではなく v1 必須。
- **しない**：ランキングの ML / n-gram 文脈（過剰）。ベースコスト＋学習＋オプション母音が v1 の高度。`IReadingEncoder`・索引は差し替え可能に保つ。
- **オプション（ゲートではない）**：lid/rid（品詞）で固有名詞・口語活用を抑制する再測定。`換えよう`/`消えゆけ` の上位はコーパス頻度ノイズで、実使用の体感より悪く見えている可能性。

## 母音絞り込み検証（2026-06-07・RomajiTrie）

`RomajiTrie`（フルローマ字格納・母音スキップ走査）で、子音のみ → フルローマ字を打ったときの候補数と正解順位（前方一致）：

| 語 | 子音のみ | フル入力 |
|----|---------|---------|
| 教育 | `kyk` 2,332件 / 18位 | `kyouiku` 109件 / **1位** |
| 共有 | `kyy` 241件 / 2位 | `kyouyuu` 30件 / **1位** |
| 許容 | `kyy` 241件 / 6位 | `kyoyou` 41件 / **1位** |
| 協議 | `kyg` 404件 / 5位 | `kyougi` 59件 / **2位** |
| 駅 | `ek` 2,056件 / 1位 | `eki` 753件 / **1位** |
| 今日 | `ky` 13,062件 / 22位 | `kyou` 3,673件 / 4位 |
| 愛 | `a` 53,031件 / 697位 | `ai` 4,900件 / 44位 |

→ **フルローマ字でほぼ 1〜4 位**。母音は 0 文字（子音のみ）〜全部まで自由に打てて、同一トライで前方一致。「子音でざっくり → 母音で絞る → 学習で定着」の 3 段構えが裏付けられた。今日（4位）・愛（44位）の残りは**学習（recency）**で吸収する設計。
